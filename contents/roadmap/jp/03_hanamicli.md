# 花見CLI

これは一見するとシンプルなクローラープロジェクトです。中核機能は、天気予報サイトから桜の予想開花日を取得し、簡単な表示ページに同期更新することです。しかし、その本当の狙いは「花見」ではなく、Agent によるツール呼び出し能力の境界を探ることにあります。私はこれのために完全な SKILL と CLI のアーキテクチャを設計しました。本質的には、これを比較実験として使いたいのです。現在の AI エコシステムにおいて、Agent が従来のツールを直接呼び出す能力は、すでに MCP（Model Context Protocol）プロトコルを置き換えられるほど強いのかを検証するためです。現在、この実験は私のローカル OpenClaw 環境でテスト実行に成功しており、この設計方針の実現可能性を証明しています。
