# SimsCommu: 音声で始まり音声で終わる会話練習の実験

SimsCommu は、マルチエージェント型の会話練習 Web アプリです。このプロジェクトの出発点はとても単純で、ひとつは私自身が口語を練習したいこと、もうひとつは STT・TTT・TTS という三段構成の音声アーキテクチャを、本当に使える AI アプリとして組み立てられるかを検証したいことです。

## プロジェクト状態：中止 · 再始動待ち

現行バージョンを中止することにしました。理由は明快です。STT -> TTT -> TTS を直列につなぐ方式よりも、Live 双方向モデルの方が優れた土台になります。リアルタイム音声対話を前提としており、双方からの割り込み、発話交代、会話の再開を自然に扱えます。これらはまさに会話練習で最も重要な体験です。

現在の実装と実験から得た知見は残しますが、このアーキテクチャへの追加投資は行いません。プロジェクトは中止し、再始動を待つ状態に入ります。再始動する場合は、既存の三段構成を拡張するのではなく、Live 双方向モデルを新しい技術的出発点とします。

## Background

私は以前から、テキストチャットの層にとどまらず、音声で始まり音声で終わる AI アプリを作りたいと考えていました。中国の ChatAI 市場では、シェアが最も大きいのは AI 性能が最も高い製品ではなく、利用者が話し、AI が返すという体験をいち早く実現した Doubao です。ATA (Audio-to-Audio) という対話形式は、今後の会話型 AI の主流になるはずです。

だから SimsCommu の背景はそれほど複雑ではありません。自分用の会話練習システムを作りたいこと、そして STT が聞き、TTT が理解と生成を担い、TTS が話すという具体的な技術ルートを検証したいことです。この流れが成立すれば、その先の多くの音声シナリオもこの土台の上で発展させられます。

## Implementation

現在の実装では、この主ループはすでにつながっています。利用者がまず話し、システムがその音声を STT に送り、次にテキストを対話層の LLM に渡して処理し、最後に TTS で返答をもう一度音声として再生します。練習をより場面会話に近づけるため、単一ロールのチャットにはせず、ホストエージェントを加えています。ホストエージェントが会話シナリオを分析し、それに対応する業務ロールを設定して、次のターンで誰が返答するべきかをシステム自身に判断させます。

構造としては、これはまだ比較的わかりやすい三段式アプリです。STT -> TTT -> TTS。率直に言えば、私がこのコードを実装した時点で、この構成はすでに少し古くなっていました。業界では STT と TTS を同時に扱う双工アーキテクチャがすでに実現されており、AI が利用者の発話を遮ることも、利用者がいつでも割り込むこともできます。

## 再始動の条件

再始動時には、Live 双方向モデルを中心に音声処理の主経路を設計し直し、その上でホストエージェント、ロール制御、練習フィードバックをリアルタイム会話へどう組み込むかを改めて検討します。
