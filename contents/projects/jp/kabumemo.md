# Kabumemo：週末 CRUD から長期的な AI 開発実験へ

Kabumemo は、Notion の代わりとなる個人株式取引ジャーナルとして始まりました。汎用ツールに合わせて自分のデータを変形し続けることに疲れ、GPT-5 Codex にハンドルを渡し、Vibe Coding で自分のワークフローに合うシステムを作ることにしました。

最初の版は本当に週末で作った CRUD プロジェクトでした。その後、Agent の能力が大きく進歩するたびに、新しい Agent にリファクタリングさせています。Kabumemo は日常的に使う道具であると同時に、AI Agent が実際のソフトウェアを長期的に保守、拡張、デプロイできるかを観察する継続的な実験になりました。

## リファクタリング後

現在の Kabumemo は静的ページではありません。Vue 3 + TypeScript のフロントエンドと FastAPI バックエンドで構成された完全なアプリケーションです。取引、保有、資金、税務の画面を持ち、売買記録、通貨・資金グループ別の保有と実現損益、未精算税金を管理できます。

資金機能は複数の JPY/USD グループ、発効日付きの資本追加、年度別パフォーマンス比較、為替換算に対応しています。選択した取引を一往復の売買として照合し、粗利益、純利益、税務影響、年率換算リターンを計算できます。保有画面では一年分の価格履歴に売買点を表示します。

JSON は確認・復旧しやすい正本として残しつつ、すべての書き込みを SQLite に同期します。インポート、整合性確認、証券会社 CSV からの再構築スクリプトにより、個人アプリでも監査可能な復旧経路を持たせています。

## AI 開発実験

このプロジェクトでは「私が要件を定義してレビューし、Agent がコードを変更する」という原則を維持しています。TypeScript、Pydantic、pytest、Vue の型チェックが品質境界です。一度でコードを生成できるかより、既存のドメインモデルを理解し、実データを壊さずにフルスタックのリファクタリングを行えるかを重視しています。

## デプロイ

本番ビルドでは Vue をコンパイルし、FastAPI が画面と API の両方を配信します。現在は Mac mini 上の Apple Container で稼働し、永続データはホストディレクトリからマウントされます。そのため、コンテナやイメージを置き換えても取引記録は失われません。

## Technical Specifications

Vue 3 / TypeScript / Vite / FastAPI / Pydantic / JSON / SQLite / ECharts / pytest / Apple Container

## 次の段階

一括インポート・エクスポート、フィルター、バックアップと復元、マルチカレンシーの境界テストを引き続き改善します。長期的には、不透明な自動化へ実取引データを渡すのではなく、明示的な許可と監査可能性のもとでローカル Agent が Kabumemo を利用できるようにします。
