# Kabumemo: 100% Vibecoding による AI 開発実験

Kabumemo は、AI のみで構築された個人向け株式取引記録サイトです。このプロジェクトでは、「手書きコードゼロ」に挑戦し、GitHub Copilot Agent を指示で導くことだけで開発を完了させました。これは単なる機能的な個人プロジェクトではなく、Copilot Agent と Claude Code（CC）の能力差を検証するためのベンチマークテストでもあります。

## Background

このプロジェクトは、Claude Code の公開初期に立ち上げられました。当時、GitHub Copilot Agent はまだテスト段階にあり、その「全自動構築」の能力には業界でもまだ疑問が残っていました。私はこのプロジェクトを通じて、ひとつの中核仮説を検証したかったのです。完全に個別化されたシステム要件に対して、現在の AI Agent はすでにアーキテクチャ設計からフルスタック実装までを独立して完遂できる能力を持っているのか。同時に、これは私にとって Copilot Agent の反復進化を長期的に観察するサンプルにもなります。

## Implementation

技術選定の面では、プロジェクトの機能自体は複雑ではないものの、私はあえて強い型制約を導入し、厳格な開発環境での AI の挙動を検証しました。

- フルスタック型ガード: フロントエンドは JavaScript ではなく TypeScript を採用し、バックエンドの Python では Pydantic を強制的に有効化しました。型システムとデータ検証モデルによって、AI が生成するコード品質をより正確に制御し、ロジックの幻覚を減らせます。

- Agent ベンチマークテスト: Copilot Agent に大きな更新が入るたびに、このプロジェクトを一度リファクタリングします。異なるバージョンのコード生成効率と Bug 率を比較することで、AI 能力の向上を定量化します。

- 手書き介入ゼロ: 全工程を通して、私は要件提示とロジックのレビューだけを担当し、コード修正、環境構成、デバッグはすべて Agent が完了します。

## Future Work

現在、私は自分専用の Local Agent を構築しており、Kabumemo はその重要なモジュールとして統合される予定です。

* CLI モード拡張: システムにコマンドラインツールを追加し、Agent が直接利用できるようにします。

* 自動入力: 取引記録の入力をローカル Agent に任せ、データ取得から保存までの完全自動フローを実現します。
