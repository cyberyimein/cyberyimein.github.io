# TeaForge: Agent AI 時代のための信頼できるテストと文書化の閉ループを構築する

Agent AI が開発に深く入り込む現在、エンジニアの生産性はかつてないほど解放されています。しかし同時に、ひとつの見えにくい課題が生まれます。それはコントロールの喪失です。AI が大量のコードを一瞬で生成するとき、エンジニアにロジックを直感的に検証する手段がなければ、「怖くて触れない、確認しづらい」という状況に陥ります。

TeaForge はそのために生まれました。これは単なるツールではなく、Agent AI のための SKILL 群と CLI でもあります。その中核的な使命は、AI がテストコードを生成すると同時に、人間が読める記述的な文書も同期生成し、AI のあらゆる判断が「根拠を追跡できる」状態にすることです。

## Background

コード生成が徐々に AI 主導になっていく中で、エンジニアの役割は「書き手」から「レビュー担当」へと変わりつつあります。直感的な文書が不足しているため、人間は AI が生成したコードのロジックの閉ループを素早く確認しにくいのが現状です。私たちに必要なのは、AI がコード作業を完了するのと同時に、「人間に説明可能な」論理的証拠も提示しなければならないようにするツールです。そうすることで、コミュニケーションコストを下げ、プロジェクトの可制御性を高められます。

## Implementation

TeaForge は、次の 3 つの側面から AI 生成コードの信頼問題を解決します。

##### 1. 業界標準に準拠した自動テストチェックリスト

TeaForge は pytest と jest をもとにテストロジックを自動抽出し、日本のソフトウェア工学標準に準拠したテストチェックリスト（PCL）を生成できます。この構造化されたチェックリストにより、AI が書いたテストケースがテスト標準を満たしているかを開発者が一目で把握でき、上位向け説明やコンプライアンス確認が容易で効率的になります。

##### 2. ロジックの可視化: Mermaid フローチャート

コード行を見るだけでは十分ではありません。TeaForge は AI に対して Mermaid 構文による対応フローチャートの生成を促します。開発者はグラフィカルな画面でロジックの流れを素早く検証でき、テストカバレッジレポートと組み合わせることで、すべての分岐経路が検証済みであることを確認できます。

##### 3. Agent-Native CLI 設計

TeaForge のコマンドラインツールは Agent との対話向けに最適化されています。--help の出力や実行失敗後の自然言語レシートによって、Agent はエラー原因をより正確に理解し、「自己思考」と修正を行えます。

## Future Work

現在、TeaForge は単体テスト段階における文書化の問題をひとまず解決しています。次の段階では、さらに次の点を探っていきます。

- レポート階層の向上: プロジェクト全体を対象にした、より高次の管理・意思決定レポートを生成すること。

- シナリオ境界の拡張: この「生成そのものが証明である」というモデルを、単体テストからより複雑な統合テストの場面へ拡張すること。
