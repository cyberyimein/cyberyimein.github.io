# BuddyNeko：Agent に猫型の身体を与える

BuddyNeko は、M5Stack CoreS3 StackChan 向けのコミュニティ Buddy デザインです。CoreS3 を猫型のデスクトップ端末に変え、高レベル AI、STT/TTS、ツール呼び出し、ポリシー判断はホスト側に残し、デバイスは軽量な Call Buddy プロトコルを通じて、表情、タッチ、カメラセンシング、音声、LED、サーボ動作を担当します。

## Agent Host から実体 Buddy へ

テキストだけを返す Agent は、デスクトップ上の相棒として体験しにくいものです。コーディング、思考中、承認待ち、完了、エラーといった状態は、表情、光、動作、タッチを通して見えるようにできます。BuddyNeko は複雑な推論を ESP32 に押し込むのではなく、デバイスをシンプルでビルドしやすく、デバッグしやすいものに保ち、モデル呼び出し、ツール実行、ポリシー判断をホスト Agent に任せます。

これにより、Anomalo のような Agent Host と実体デバイスの境界も明確になります。ホストは AI、STT、TTS、ツール、認可ポリシーを持ち、StackChan は状態を表現し、タッチ、音声、センサーイベントをホストへ返します。

## 現在の実装

- CoreS3 の画面向けに 16 色の猫型スプライトアニメーションを搭載。現在のファームウェア識別バージョンは `0.7.0`。
- 115200 baud の USB Serial と、オプションの TCP テキスト/イベント転送。
- Call Buddy コマンドによる idle、listening、thinking、speaking、coding、approval、done、error 状態。
- タッチ操作、承認レスポンス、サーボ移動、RGB エフェクト、カメラ追従、オプションのホスト視覚エンドポイント。
- 半二重 PCM16 によるマイク・スピーカー転送。音声処理はホスト側で実行します。
- アセット生成、ディスプレイキャプチャ、開発ホストのリース維持を行う、依存関係のない Python ツール。

## Call Buddy と安全境界

プロトコルは、行単位のテキストコマンドと JSON Lines イベントを使用します。まず USB Serial で動作し、同じモデルを TCP Agent 接続でも利用できます。開発モードへの切り替えには USB Serial から短期リースを明示的に要求し、継続的に更新する必要があります。USB を電源につないだだけで開発ホストへ自動切り替えすることはなく、リースが期限切れになるとデプロイ先ホストへ戻ります。

デバイスは承認の意思だけを取得し、最終的な認可はホストのポリシーが行います。サーボ動作もファームウェア側の安全範囲に収めます。この境界により、物理デバイスを素早く改良しながら、安全判断を観測できないハードウェア操作の中に隠さずに済みます。

C++ / Arduino / ESP32-S3 / M5Stack CoreS3 / StackChan BSP / Python / USB Serial / TCP / PCM16

## プロジェクト状態

BuddyNeko は現在活発に開発中で、実際の CoreS3 StackChan ハードウェア向けにファームウェアをコンパイルできます。M5Stack 公式出荷ファームウェアではなく、コミュニティによる実体 Buddy デザインです。今後は独立した Agent Host との接続をさらに整え、デスク上の表情、動作、音声、承認を Agent ワークフローの一部にしていきます。
